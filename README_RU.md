# Update CLI Tools

Простой и эффективный bash-скрипт для обновления различных инструментов командной строки (CLI), включая менеджеры пакетов и агенты LLM CLI.

## Особенности

- Автоматическое обнаружение установленных инструментов в вашем PATH
- Обновление нескольких инструментов за одну операцию
- Подробное логирование с опциональным ротированием логов
- Простая установка одной командой

## Как это работает

Скрипт проходит по предопределённому списку инструментов. Для каждого инструмента он проверяет наличие исполняемого файла в системном PATH. Если файл найден, скрипт выполняет соответствующую команду обновления.

## Поддерживаемые инструменты

### Менеджеры пакетов
- `npm` - менеджер пакетов Node.js
- `bun` - быстрый JavaScript-рантайм и менеджер пакетов
- `composer` - менеджер зависимостей PHP

### Агенты LLM CLI
- `gemini` - CLI для Google Gemini
- `qwen` - CLI для Qwen Code от Alibaba Cloud
- `copilot` - CLI для GitHub Copilot
- `codex` - CLI для OpenAI Codex
- `claude` - CLI для Anthropic Claude Code
- `cursor-agent` - агент для IDE Cursor
- `goose` - помощник разработчика Goose
- `vibe` - CLI для Mistral AI Vibe

## Предварительные требования

- Bash shell
- Подключение к интернету для загрузки обновлений
- Соответствующие права для обновления установленных инструментов
- `curl` для удаленного выполнения скрипта

## Использование

### Удаленное выполнение (рекомендуется)
Выполнение напрямую из GitHub без клонирования:
```bash
curl -fsSL https://raw.githubusercontent.com/kas-cor/update-cli-tools/refs/heads/main/run.sh | bash
```

### Локальное выполнение
Клонируйте репозиторий и запустите скрипт локально:
```bash
git clone https://github.com/kas-cor/update-cli-tools.git
cd update-cli-tools
bash run.sh
```

## Настройка удобного псевдонима (alias)

Для простого доступа добавьте псевдоним в файл конфигурации вашего shell (например, `~/.bashrc`, `~/.zshrc`):
```bash
alias update-tools='curl -fsSL https://raw.githubusercontent.com/kas-cor/update-cli-tools/refs/heads/main/run.sh | bash'
```

После добавления псевдонима и перезагрузки конфигурации shell (`source ~/.bashrc`), вы можете просто выполнить `update-tools` для обновления ваших инструментов.

## Логирование

Скрипт записывает свои действия в `~/.update-cli/update-cli.log`. Ротация логов автоматически осуществляется через `logrotate`, если он доступен в вашей системе, сохраняя до 7 дней логов.

## Вклад в проект

Мы приветствуем вклад в расширение списка поддерживаемых инструментов!

### Добавление нового инструмента

Чтобы добавить новый инструмент в список обновлений:

1. Сделайте fork репозитория
2. Добавьте новую запись в массив `tools` в файле `run.sh`
3. Формат: `"имя_исполняемого_файла:команда_обновления"`
4. Отправьте pull request

#### Пример добавления
```bash
"newtool:newtool update --latest"
```

### Руководящие принципы для новых инструментов

- Убедитесь, что инструмент широко используется в процессах разработки
- Предоставьте надежную команду обновления
- Проверьте, что команда обновления работает как ожидается
- Следуйте тому же формату, что и существующие записи
